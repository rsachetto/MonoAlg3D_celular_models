Installation & Publishing Guide

Quick setup guide to integrate the MonoAlg3D Plugin Manager and contribute models.

How to install plugins

  1. Configure CLI Tool

    To point the local plugin manager to this registry, configure it in your MonoAlg3D directory:

    python scripts/pm.py config repo_owner sachetto
python scripts/pm.py config repo_name monoalg3d-plugins-repo
python scripts/pm.py config branch main

  2. List Remote Models

    Query models available on the registry:

    python scripts/pm.py list

  3. Install a Model

    Install the cellular model plugin to your local project:

    python scripts/pm.py install fhn_mod

    This fetches files and adds them under src/models_library/<model_id>/

  4. Rebuild the Simulator

    Re-run the main build script. The build system automatically scans and compiles the new plugin as a shared library:

    ./build.sh

Publishing / Contributing

You can upload your own cellular models directly via the CLI tool. This creates a Pull Request on this repository automatically.

How to Upload:

  1. Ensure your model is placed inside src/models_library/your_model_id/ with a valid plugin.json metadata file.

  2. Run the CLI publish command:

    python scripts/pm.py publish your_model_id

  3. Provide your GitHub Personal Access Token (PAT) when prompted. The tool handles cloning, staging, and creating the PR using your account.

C/C++ Files Only: We host only source code files (C/CUDA) and compile them dynamically on execution. Binary files are not accepted.
